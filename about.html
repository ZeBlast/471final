Project Documentation

README

After Graduation is an interactive D3 webpage about post-college outcomes. It combines College Scorecard financial data with Post-Secondary Employment Outcomes (PSEO) earnings data to compare colleges, states, debt, cost, and earnings across multiple graduation cohorts.

The website is organized into three progressive sections: Learn (state-level trends), Explore (college map), and Decide (direct college comparison). Together they tell the story of which colleges pair strong first-year earnings with lower debt and lower yearly cost, a question every college applicant should be asking.

Datasets

schools.csv provides institution identity and location data sourced from Kaggle / College Scorecard. Fields used include name, city, state, lat, lon, tuition, room cost, and book/supply cost. This file powers the college map and geographic placement.

aid.csv provides borrowing and debt metrics from Kaggle / College Scorecard. Fields used include loan_principal, pell_grant_rate, federal_loan_rate, students_with_any_loan, median_debt_suppressed, plus parsed completer debt and monthly payment fields.

pseo_all_institutions.csv is the PSEO institution lookup. It provides the institution name and ID side of the merge and is matched to Scorecard schools by state and institution name.

pseoe_all.csv provides earnings outcomes by cohort from the U.S. Census Bureau LEHD PSEO experimental data. Filtered to inst_level = I, cip_level = A, degree_level = 05. Cohorts include the aggregate 0000 plus 2001, 2004, 2007, 2010, 2013, 2016, and 2019. Note that year-5 and year-10 earnings are only populated for certain cohorts, so some dropdown selections may return fewer states or schools.

combined_pseo_all_cohorts.csv is the main runtime dataset. It merges school identity, tuition, debt, aid metrics, and cohort earnings into a single file. Each row represents one institution for one graduation cohort.

How the Visualizations Use the Data

Section 1 - Learn: Aggregates data by state within the selected cohort and compares earnings, debt, cost, Pell grant rate, and earnings-to-debt ratio.

Section 2 - Explore: Plots institutions on a US map using latitude and longitude. Colors and sizes college dots by the chosen metric. Supports zoom, pan, and click-to-compare.

Section 3 - Decide: Lets the user compare schools saved from the map. Charts median earnings at Year 1 versus the completer debt baseline for each saved college.

Important Notes

  • The app reads combined_pseo_all_cohorts.csv directly in the browser with D3.
  • The dataset does not contain cohorts after 2019.
  • Year-5 and year-10 earnings are not available for all cohorts. Some combinations will return fewer results.
  • Data sources: Kaggle College Scorecard and U.S. Census Bureau LEHD PSEO.

Transparency

AI Tool Usage

This project used AI tools to accelerate development. Below is a transparent log of where and how AI assistance was applied, consistent with course policy on AI usage disclosure.

Code

D3 chart scaffolding: AI was used to generate initial D3 chart templates for the bar chart, scatter/dot chart, and line chart. All chart logic was then customized to fit the PSEO dataset structure, metric labeling system, and interactive state management used in the project.

Debugging

Dropdown interaction bugs: AI was consulted to trace bugs in the dropdown controls and state-filter selects that caused empty chart states. The fix involved restructuring how available metrics are determined per cohort and improving user feedback when data is sparse.

Data

Python merge script: AI helped draft the initial structure of build_combined_pseo.py, the script that joins College Scorecard and PSEO data. All merge logic, field selection, and output column naming were reviewed and adjusted by the team to match the actual CSV schemas.

Writing

Story narrative text: AI assisted in drafting the introductory hero paragraph and section callout boxes to help frame the visualization as a narrative journey rather than a plain dashboard. All text was reviewed and edited by the team.

UX

CSS layout and responsive design: AI was used to generate initial CSS custom property definitions and responsive media query scaffolding. All final design decisions including colors, spacing, border radii, and panel layout were made and refined by the team.

Review

Peer feedback incorporation: AI helped consolidate and prioritize peer review feedback and suggested implementation approaches for adding zoom controls, tooltips, story text, and this about page. The team applied and verified all changes.
